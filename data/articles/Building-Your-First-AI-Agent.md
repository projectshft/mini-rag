Building Your First AI Agent

Last year I joined an AI start-up and started building my first agent for a large project we were working on.

It was exciting and messy. No one on the engineering team had any expertise with AI or working with LLMs.

This “agent” was supposed to take a prompt, determine if we could actually handle the request and then do a search of either a vector database or do a web search. It might return some text or an array of objects depending on the outcome.

It was bloated, inefficient and costly.

I had no clue what I was doing.

This massive agent was breaking one of the cardinal rules of building software: SRP.

Single Responsibility Principle.

Our CEO was an AI OG, luckily, and had some experience with agents before they became cool.

He helped us re-think our logic and then we broke up our “agent” in to multiple, self-contained agents that were each responsible for 1 thing.

1 agent to summarize the conversation.

1 orchestrator to decide which agent to call next.

1 agent for a vector search.

1 agent for a web search.

Clean. Simple. Cost-effective. Composable.

While everyone is having an existential crisis about the future of tech — there is a massive re-shuffling happening under our feet. This means massive opportunity.

The more I work with AI, the more bullish I am about the future of tech and software engineering specifically.

Most of the same skills that were important before AI will be important going forward: debugging, system design, clean code and solving tough problems with code aren’t going anywhere.

Owning a camera doesn’t make you a photographer, much in the same way that writing some code with AI does not make you a software developer.

While the “old” skills are still relevant, the tech we use is undergoing a massive overhaul. Learning how to build agents, work with vector databases and monitor these agents are emerging skills that most developers just haven’t had experience with yet.

This shift is why I’m building something for Parsity students to make sure they have exposure to this new tech stack. You can check it out at Parsity.ai

Because I like you, I built a very small project to introduce you to agents using Gemini and plain HTML and JS.

This project uses 2 agents to estimate how many calories you consumed based on what you ate in the day.

One agent determines if we have enough information to provide an estimate of your calories and the other agent returns the estimate.

Simple and maybe even useful.

You won’t get much out of just reading about it though. The ONLY way to learn is to get your hands dirty.

Step 1: Get a free key from Google to use their Gemini model: https://aistudio.google.com/apikey

Step 2: Visit https://codepen.io/Brian-Jenney/pen/PwqLQLW?editors=1010 and start getting your hands dirty with the Diet Tracker App.

Step 3: Break, re-write and extend the project. Change the output format. Can you add another agent? The only way to actually learn is to get your hands dirty.

If the code is out of your depth then you should consider our Dev30 program to help you learn the basics of JS.

Enjoy!
