Sparse vs Dense Vectors

Two ways to search, one powerful combination

The Search Problem

You have a query: "How to prevent re-renders in React?"

Should match: "Optimizing React performance with memoization"

Should also match: "Using React.memo to prevent re-renders"

Should NOT match: "How to render 3D graphics in React"

How do we find documents by meaning AND exact terms?

Side by Side

Dense (Semantic)

  • Finds similar meanings
  • Works across languages
  • Handles paraphrasing
  • May miss exact terms
  • Less interpretable

Sparse (Keyword)

  • Exact term matching
  • Preserves terminology
  • You know why it matched
  • Misses synonyms
  • Struggles with typos

Hybrid Search: Best of Both

Dense
+
Sparse

Hybrid

Query: "How to prevent re-renders in React?"

Dense finds: docs about React performance, optimization patterns

Sparse finds: docs with "re-render", "React.memo", "useMemo"

Hybrid: Combines both, weighted by alpha parameter

Pinecone Implementation

// Upsert with both vector types
await index.upsert([{
  id: 'doc-1',
  values: [0.12, 0.45, 0.23, ...], // Dense
  sparseValues: {
    indices: [142, 892, 1203],     // Word positions
    values: [0.8, 0.6, 0.4]         // Word weights
  },
  metadata: { text: '...' }
}]);
// Query with hybrid search
const results = await index.query({
  vector: denseEmbedding,
  sparseVector: sparseVector,
  topK: 10,
  alpha: 0.5  // 0 = sparse only, 1 = dense only
});

When to Use What

Use Hybrid When:

  • Technical documentation
  • Product catalogs with IDs
  • Legal/medical terms matter
  • Users search exact phrases

Dense-Only is Fine When:

  • You're just starting out
  • Content is conversational
  • No critical exact-match needs
  • Simplicity > perfection

Start simple. Add hybrid when you see exact-match failures.
